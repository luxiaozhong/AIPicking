# 超跌反弹策略（Oversold Bounce）分析与翻译

> 将外部系统的 `oversold_bounce.py` 翻译为本后端回测框架可执行的策略。
>
> 源文件：[oversold_bounce.py](../oversold_bounce.py)（原始）→ [oversold_bounce.py](../backend/app/strategies/examples/oversold_bounce.py)（翻译版）

---

## 一、策略概述

**科技股超跌反弹**量化选股策略，核心思路是捕捉 **"放量急跌 → 缩量止跌 → 放量反弹"** 的三阶段信号链。

- **适用范围**：创业板（300/301）+ 科创板（688/689）
- **策略类型**：纯量价技术面，不涉及基本面
- **信号频率**：每日收盘后批量扫描
- **输出**：按评分降序排列的 Top 10 推荐，附带详细诊断信息

---

原始脚本使用**类继承 + 装饰器注册**模式：

```

        # 2. 查数据库筛选股票池
        # 3. 每只股票执行三阶段检测
        # 4. 按换手率降序排列返回
```

### 数据依赖

| 表 | 字段 | 用途 |
|----|------|------|
| `Stock` | `code`, `name`, `list_date`, `is_st` | 股票池过滤 |

| `DailyPrice` | `open/high/low/close/volume/turnover_rate` | OHLCV + **预计算**换手率 |

## 三、三阶段信号链详解

这是策略的灵魂。每个阶段都是**硬过滤**——不满足就直接淘汰该股票。

### 阶段一：急跌识别

在目标日当天的回溯窗口（默认 20 天）内检查：

| # | 条件 | 参数（默认值） | 逻辑 |
|---|------|---------------|------|
| 1a | 找高点 | `lookback_days=20` | 20 日内最高收盘价 |
| 1b | 回撤幅度 | `drawdown_pct=15%` | `(peak - current) / peak × 100 ≥ 15%` |
| 1c | MA20 偏离 | `ma20_deviation=5%` | 收盘价低于 MA20 至少 5%（排除均线上方的正常回调） |
| 1d | 恐慌放量 | `panic_vol_ratio=2.0` | 窗口内**任意一天**成交量 > 20日均量 × 2.0 |

> **关键设计**：恐慌放量不要求发生在最高点那天，窗口内任意一天出现过即可。更符合"放量下跌"可能在回撤中段出现的实际情况。

### 阶段二：缩量止跌

在买入日前 `stabilize_window`（默认 3）天内**从近到远**搜索止跌日：

| # | 条件 | 参数（默认值） | 逻辑 |
|---|------|---------------|------|
| 2a | 缩量 | `shrink_vol_ratio=0.6` | 止跌日量 < 20日均量 × 60% |
| 2b | 缩量递减 | — | 止跌日量 < 前一日量（确认缩量趋势） |
| 2c | 不再创新低 | `low_tolerance=0.04` | 止跌日最低 ≥ 前日最低 × 96%（允许 4% 容差） |

> **关键设计**：`low_tolerance` 允许最低价略微下探，避免过于严格错过"双底"或"平底"形态。找到第一个满足条件的就 break，优先取最近的止跌日。

### 阶段三：反弹触发

对比买入日与止跌日：

| # | 条件 | 参数（默认值） | 逻辑 |
|---|------|---------------|------|
| 3a | 放量反弹 | `bounce_vol_ratio=1.2` | 买入日量 > 止跌日量 × 1.2 |
| 3b | 收盘强势 | `close_upper_ratio=0` | 0=不检查；设为 0.5+ 可过滤尾盘跳水的假反弹 |

## 四、翻译变更

### 4.1 架构适配

| 维度 | 原始（外部系统） | 翻译版（本后端） |
|------|-----------------|-----------------|
| 接口 | `class OversoldBounceStrategy(BaseStrategy).screen()` | `run(data: dict) -> list[dict]` |
| 数据获取 | 直接查询 PostgreSQL（`SessionLocal()`） | 回测引擎切片后传入 `data` dict |
| 均线来源 | `DailyIndicator.ma20` / `.vol_ma20`（预计算） | `df["close"].rolling(20).mean()`（动态计算） |
| 换手率 | `DailyPrice.turnover_rate`（预计算） | `_compute_turnover()` 自适应检测 vol 单位后计算 |
| 股票代码 | `Stock.code` + `startswith("300")` | `ts_code` + `startswith("300")`（含 `.SZ`/`.SH` 后缀） |
| 注册机制 | `@register` 装饰器 | `run(data)` 函数由 `exec()` 加载 |
| 大盘择时 | 单独查询 `DailyIndicator` 表 | 从 `data["daily"]["399006.SZ"]` 直接计算 |

### 4.2 评分体系变更

原始评分很简单：`换手率 × 100`。但这过度依赖换手率，忽略了回撤深度、缩量程度等其他维度的信息量。

翻译版采用**多维度复合评分**（满分 100+10）：

| 维度 | 分值 | 逻辑 |
|------|------|------|
| 回撤幅度 | 0-30 | ≥25%→30, ≥20%→24, ≥15%→18, else→10 |
| MA20 偏离 | 0-20 | ≥10%→20, ≥7%→15, ≥5%→10, else→5 |
| 缩量程度 | 0-20 | <0.3→20, <0.5→15, <0.6→10, else→5 |
| 反弹放量强度 | 0-20 | ≥2.5→20, ≥2.0→16, ≥1.5→12, else→6 |
| 收盘位置 | 0-10 | 仅当 `close_upper_ratio > 0` 时生效 |
| 换手率（叠加） | 1-10 | ≥8%→+10, ≥5%→+8, ≥3%→+6, ≥1%→+4, else→+1 |

### 4.3 大盘择时

原始逻辑保留，但实现方式变为直接计算：

## 五、数据质量问题与解决方案

### 5.1 vol 单位不一致

**发现**：同一数据库的 `daily.vol` 字段在不同交易所/不同日期使用了不同单位：

| 板块 | `amount/(close×vol)` 比率 | vol 单位 |
|------|--------------------------|----------|
| 创业板（300/301） | ≈100 | 手（lots, ×100 股） |
| 科创板（688/689） | ≈1 或 ≈100（混用） | 股 或 手 |

**解决方案**：`_compute_turnover()` 函数自适应检测比率，ratio>50 判定为"手"并自动 ×100 归一化。同时设 100% 上限防止极端异常值。

### 5.2 系统性数据异常日（如 20260527）

**发现**：2026-05-27 科创板整体成交量骤降 200 倍（中位数从 500 万 → 2.4 万），大量股票出现接近零成交的"地量日"，被策略误识别为止跌日，产生 200x~800x 的荒谬反弹放量比。

**原始策略无此问题**的原因：上游 `DailyIndicator` 表数据质量有保证，异常日被预过滤。

**解决方案**：在阶段二增加两道数据完整性校验（不影响原始策略语义）：

1. **amount 校验**：`amount ≥ close × vol × 0.1`，排除 amount 字段损坏（如 688012 中微公司 amount=36500 vs 理论值约 5 亿）
2. **vol 下限**：`vol ≥ vol_ma20 × 1%`，排除系统性异常日（正常缩量如 30% 均量完全不受影响）

这两项校验显著提高了信号质量，反弹放量比从 200x~800x 回归到 1.2x~4.8x 的合理范围。

### 5.3 list_date 不可用

后端 `_load_data()` 的 `stocks` 查询未包含 `list_date` 字段，策略使用 `MIN_HISTORY=60`（最低 60 个交易日）作为等效过滤。

## 六、策略执行流程

```
run(data)
  │
  ├─ 1. 参数覆盖（config dict）
  │
  ├─ 2. 大盘择时（可选）
  │     └─ 创业板指收盘 < MA20 × (1 - N%)? → no → 返回空
  │
  ├─ 3. 遍历 data["daily"] 中的每只股票
  │     ├─ 板块过滤：300/301/688/689
  │     ├─ ST 过滤：名称含 "ST"
  │     ├─ 市值过滤：20亿 ~ 5000亿
  │     ├─ 数据量：≥60 个交易日
  │     │
  │     ├─ check_oversold_bounce(df)
  │     │   ├─ 阶段一：急跌识别
  │     │   ├─ 阶段二：止跌日搜索（含数据完整性校验）
  │     │   ├─ 阶段三：反弹确认
  │     │   └─ 打分 + 信号文字
  │     │
  │     └─ 换手率估算 + 加分
  │
  └─ 4. 按 score 降序，返回 Top 10
```

---

## 七、参数清单

共 12 个可调参数（可通过 `config` dict 覆盖）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `drawdown_pct` | 15.0 | 回撤幅度阈值（%） |
| `lookback_days` | 20 | 找高点+恐慌量的回溯窗口（天） |
| `panic_vol_ratio` | 2.0 | 恐慌放量倍数 |
| `ma20_deviation` | 5.0 | 收盘价需低于 MA20 的幅度（%） |
| `shrink_vol_ratio` | 0.6 | 缩量标准：止跌日量 < 20日均量 × N |
| `stabilize_window` | 3 | 止跌日搜索窗口（天） |
| `low_tolerance` | 0.04 | 止跌低点容差 |
| `bounce_vol_ratio` | 1.2 | 反弹放量倍数 |
| `close_upper_ratio` | 0.0 | 收盘在振幅上半区（0=不检查） |
| `min_list_days` | 60 | 最低上市天数（已被 MIN_HISTORY 替代） |
| `market_timing` | True | 是否启用大盘择时 |
| `market_ma20_below` | 1.5 | 指数收盘需低于 MA20 的幅度（%） |

## 八、输出字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `ts_code` | string | 股票代码（含后缀，如 `300001.SZ`） |
| `name` | string | 股票名称 |
| `score` | int | 综合评分（0-110） |
| `signal` | string | 信号描述文字，如 "回撤25.9%→恐慌放量→缩量至0.46x均量→反弹放量1.5倍 换手6.1%" |
| `breakdown` | dict | 评分明细 `{drawdown, ma20_dev, shrink, bounce, close_pos}` |
| `details` | dict | 完整诊断信息（回撤、MA20偏离、换手率、缩量比、反弹比、止跌日、3/5/10日涨跌幅等） |
